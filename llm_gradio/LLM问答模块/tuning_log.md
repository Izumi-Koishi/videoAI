# 回答效果调优记录

## 调优目标
- 减少 LLM 幻觉（编造不存在的检测结果）
- 提高回答的相关性和准确性
- 优化用户体验和交互流程

## 调优策略

### 1. Prompt 工程优化
| 策略 | 说明 | 状态 |
|------|------|------|
| 系统角色设定 | 明确定义为"基于检测结果回答的助手"，限制自由发挥 | ✅ 已实施 |
| Few-shot 示例 | 提供 2 个标准问答示例，规范输出格式 | ✅ 已实施 |
| 反幻觉指令 | 明确要求"不确定时说明"、"严格基于上下文" | ✅ 已实施 |
| 输出格式约束 | 三段式结构：直接回答→详细分析→置信度评估 | ✅ 已实施 |
| 引用强制要求 | 要求引用检测目标ID和相似度 | ✅ 已实施 |

### 2. 检索质量优化
| 策略 | 说明 | 状态 |
|------|------|------|
| 相似度阈值过滤 | 过滤相似度 < 0.3 的低质量检索结果 | ✅ 已实施 |
| Top-K 可调 | 用户可调整检索数量，平衡精度和召回 | ✅ 已实施 |

### 3. 后处理优化
| 策略 | 说明 | 状态 |
|------|------|------|
| 幻觉关键词检测 | 检测"一般说来"、"通常情况下"等模糊推断表述 | ✅ 已实施 |
| 置信度提取 | 自动从回答中提取置信度标识 | ✅ 已实施 |
| 引用目标追踪 | 统计回答中引用的目标ID，验证一致性 | ✅ 已实施 |
| 质量警告附加 | 在回答末尾附加检测到的质量问题 | ✅ 已实施 |

### 4. 生成参数优化
| 参数 | 推荐值 | 说明 |
|------|--------|------|
| temperature | 0.3 | 较低温度减少随机性，降低幻觉概率 |
| top_p | 0.9 | 适中的核采样范围 |
| max_tokens | 1024 | 足够覆盖完整的三段式回答 |
