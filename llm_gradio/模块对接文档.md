## 二、模块二已提供的接口

### 2.1 LLM 引擎 — `llm_engine.py`

```python
from llm_engine import LLMEngine, PROMPT_TEMPLATES
```

#### 初始化参数

- `backend`：后端类型，`"openai"` 或 `"ollama"`，默认 `"openai"`
- `api_key`：OpenAI 兼容 API 的密钥，默认空
- `api_base`：API 基础地址，默认 `"https://api.openai.com/v1"`
- `model_name`：模型名称，默认 `"gpt-3.5-turbo"`
- `ollama_model`：Ollama 模型名称，默认 `"qwen2.5:7b"`
- `ollama_url`：Ollama 服务地址，默认 `"http://localhost:11434"`

#### 核心方法

```python
# 检查是否就绪
engine.is_ready() -> bool

# 生成回答（主入口）
engine.generate(
    question: str,       # 用户问题
    context: str,        # 检索到的上下文
    template_name: str,  # Prompt模板名，"auto"自动选择
    max_tokens: int,     # 最大生成token数，默认1024
    temperature: float   # 生成温度，默认0.7
) -> str                # 返回LLM回答文本
```

#### Prompt 模板库

```python
PROMPT_TEMPLATES = {
    "default":      "通用问答模板",
    "object_count": "目标统计模板（多少/几个/统计）",
    "scene_desc":   "场景描述模板（描述/场景/发生什么）",
    "technical":    "技术分析模板（置信度/精度/误检）"
}
```

每个模板都包含 `{context}` 和 `{question}` 两个占位符。`template_name="auto"` 时根据问题关键词自动匹配模板。

### 2.2 向量数据库 — `vector_store.py`

```python
from vector_store import VectorStore
```

#### 核心方法

```python
vs = VectorStore(persist_dir="./vector_db")
vs.is_ready()  # 检查是否就绪

# 存储检测结果
vs.add_detection_results(
    video_id: str,            # 视频唯一标识
    detection_results: list,  # YOLO检测结果列表（见下方数据格式）
    detection_summary: str    # 检测摘要文本
)

# 检索（返回格式化结果列表）
vs.search(video_id: str, query: str, top_k: int = 5)
-> List[Dict]  # 每项: {"document": str, "metadata": dict, "distance": float}

# 便捷方法：直接获取上下文+图片
vs.get_context_and_images(video_id: str, query: str, top_k: int = 5)
-> Tuple[str, List[str]]  # (上下文文本, 匹出图片路径列表)
```

from yolo_detector import YOLODetector

det = YOLODetector(model_name="yolov8n.pt")

det.process_video(

det.generate_detection_summary(detection_results) -> str

## 三、模块一需要完成的4项任务及对接方式

### 任务1：完成大语言模型对接

**对接方式：替换或扩展 `LLMEngine` 类**

- **接入新 LLM 后端**（如本地 transformers、其他 API）：在 `LLMEngine` 类中添加新的 `backend` 分支，仿照 `_generate_openai` 或 `_generate_ollama` 写一个 `_generate_xxx` 方法，然后在 `generate` 方法中增加路由分支。

- **替换为远程微服务**：在 `_generate_xxx` 中用 `requests.post` 调用即可。

**关键约束：`generate(question, context, template_name, ...)` 方法签名必须保持不变。**

### 任务2：设计领域专属 Prompt 模板

**对接方式：扩展 `PROMPT_TEMPLATES` 字典**

```python
# 在 llm_engine.py 中添加
PROMPT_TEMPLATES["medical"] = """你是一个医疗影像分析助手...
## 视频检测结果
{context}
## 用户问题
{question}
请给出你的回答:"""
```

**必须遵守**：
1. 模板中必须包含 `{context}` 和 `{question}` 两个占位符
2. 如需支持"auto"自动选择，在 `_select_template` 方法中添加关键词匹配

添加后 Gradio 界面的 Prompt 模板下拉框会自动包含新模板。

### 任务3：实现完整问答主逻辑

**对接方式：已实现，位于 `app.py` 的 `ask_question` 函数**

```
ask_question() 执行流程：
1. 校验输入（问题、video_id）
2. VectorStore.get_context_and_images() → 向量检索
3. LLMEngine.generate() → Prompt拼接 + LLM生成
4. 返回 (answer, context_text, image_paths)
```

如需修改问答逻辑（增加多轮对话、重排序等），直接修改 `ask_question` 函数，**返回值格式必须保持 `(str, str, List[str])`**。

### 任务4：调优回答质量，减少幻觉

- **Prompt 调优**：修改 `PROMPT_TEMPLATES` 中的模板文本，增加约束性指令
- **检索调优**：修改 `VectorStore.search` 的 `top_k` 参数，或更换嵌入模型
- **后处理**：在 `ask_question` 中对 LLM 返回的 `answer` 做截断、格式化、幻觉检测

## 五、模块一需要修改的文件清单

- `llm_engine.py` — 添加新 LLM 后端 / 添加新 Prompt 模板 / 调优模板内容（任务1、2、4）
- `vector_store.py` — 更换嵌入模型 / 调整检索策略（任务4）
- `app.py` — 修改 `ask_question` 问答逻辑 / 调整界面参数（任务3）

**不要改的**：`yolo_detector.py`（YOLO 检测逻辑已稳定）、`app.py` 中的 Gradio 组件定义和事件绑定（除非需要新增/删除界面元素）。

## 六、快速对接验证步骤

1. 启动服务：`python app.py`
2. 打开浏览器访问 `http://localhost:7860`
3. 在顶部"模型配置"中填写 API Key 或切换到 Ollama
4. 上传一个视频 → 点击"开始检测处理" → 确认检测摘要正常显示
5. 输入问题 → 点击"提问" → 确认回答、上下文、图片都正常输出
6. 以上5步都通过，对接成功
